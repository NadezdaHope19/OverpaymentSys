## 1. Введение
 
Система предназначена для учёта переплат получателей, фиксации платежей, автоматического пересчёта остатка долга и отслеживания судебных процессов. Доступ к функциям определяется ролью пользователя.
 
Основные возможности:
 
* Ведение карточек задолженностей с информацией о должнике и переплатах
* Внесение платежей с автоматическим пересчётом долга
* Отслеживание судебных дел (иск, решение, исполнительный лист)
* Формирование отчётов и экспорт данных
* Разграничение доступа по ролям
 
 
## 2. Вход в систему
 
### 2.1. Аутентификация через домен (Active Directory)
 
1. Откройте браузер и перейдите по адресу: https://[адрес_системы]
2. Введите доменные учётные данные:
   * Логин: ДОМЕН\логин (например, COMPANY\ivanov)
   * Пароль: ваш доменный пароль
3. Нажмите кнопку «Войти»
 
 
### 2.2. Выход из системы
 
Нажмите на своё имя в правом верхнем углу → «Выйти».
 
 
## 3. Общий интерфейс
 
После входа вы видите рабочую страницу, состоящую из 3 частей:

* Верхняя панель — название системы, имя пользователя и его роль.
* Навигационное меню — доступные разделы.
* Рабочая область — списки, формы, отчёты.
 
 
## 4. Блок первичной информации о переплате
 
Доступ: всем ролям только на просмотр, оператору — полный цикл кроме удаления, руководителю - полный цикл, включая удаление.
 
### 4.1. Просмотр списка должников
 
1. Перейдите в раздел «Должники».
2. Вы увидите таблицу со столбцами: СНИЛС, ФИО, Дата постановки на учет, Место работы, Сумма переплаты, Период переплаты, Источник переплаты, Причина переплаты, Остаток переплаты на текущую дату.
   * Чёрная сумма остатка — долг (активен).
   * Красная сумма остатка — излишек (необходимо вернуть плательщику)
   * Зеленая сумма остатка - долг погашен.
3. Для поиска используйте строку «Поиск» или фильтр по столбцу (по ФИО или СНИЛС).
4. Фильтры:
   * доступны на всех активных столбцах
5. Нажмите «Применить» для фильтрации, «Сбросить» для очистки.
6. Для сортировки кликните по заголовку столбца.
 
### 4.2. Создание нового должника
 
Доступ: Оператор, Руководитель.
 
1. Нажмите кнопку «Новый должник».
2. Заполните обязательные поля:
   * СНИЛС (формат: 999-999-999 99, проверка контрольной суммы)
   * ФИО
   * Дата постановки на учёт (по умолчанию текущая дата)
   * Источник переплаты (выпадающий список)
   * Причину переплаты (выпадающий список)
4. При необходимости заполните:
   * Место работы
   * Сумма переплаты (положительное число, точность до 2 знаков после запятой)
   * Период переплаты (формат ДД.ММ.ГГГГ-ДД.ММ.ГГГГ)
5. Нажмите «Сохранить».
 
Внимание: Остаток переплаты на текущую дату после создания карточки равно сумме переплаты. После внесения платежей автоматически пересчитывается. Ручное редактирование запрещено.
 
### 4.3. Редактирование карточки должника

Доступ: Оператор, Руководитель.
 
1. В списке должников кликните по строке с нужным ФИО.
2. Откроется карточка с вкладками:
   * Основная информация — все поля должника.
   * История платежей — список всех платежей.
3. Для редактирования нажмите кнопку «Редактировать» (доступно ролям: оператор, руководитель).
4. Внесите изменения и нажмите «Сохранить».
 
### 4.4. Удаление должника
 
Доступ: только Руководитель
 
1. Откройте карточку должника.
2. Нажмите «Удалить».
3. Подтвердите действие. Удаление приведёт к удалению всех связанных платежей и судебных дел.
 
 
## 5. Блок платежной информации
 
Доступ: оператор, руководитель (просмотр — всем).
 
### 5.1. Внесение платежа
 
1. Перейдите в карточку должника.
2. На вкладке «История платежей» нажмите «Внести платёж».
3. Заполните форму:
   * Дата платежа (по умолчанию сегодня)
   * Сумма (обязательно, положительное число)
   * Источник переплаты (выбрать из выпадающего списка)
4. Нажмите «Сохранить».
 
Что произойдёт автоматически:
 
* Сумма долга уменьшится на величину платежа.
* Если платёж превышает оставшийся долг, остаток становится отрицательным — это излишек. Поле «Остаток переплаты» покажет сумму, которую необходимо вернуть должнику. Программа предложит сформировать уведомление должнику.
 
### 5.2. Редактирование и удаление платежа
 
Доступ: Оператор (редактирование), Руководитель (редактирование и удаление).
 
1. В истории платежей найдите нужный платёж.
2. Нажмите «Редактировать» (карандаш) или «Удалить» (корзина).
3. При изменении суммы или удалении платежа остаток долга пересчитается автоматически.
 
 
## 6. Юридический блок
 
Доступ: Юрист, Руководитель, Администратор (просмотр — всем).
 
### 6.1. Создание судебного дела
 
1. Откройте карточку должника.
2. Перейдите на вкладку «Судебное дело».
3. Нажмите «Новое дело».
4. Заполните:
   * Дата направления искового заявления (обязательно)
   * Наименование суда
5. Нажмите «Сохранить».
 
### 6.2. Внесение результатов рассмотрения
 
1. Откройте судебное дело (клик по строке в списке).
2. Нажмите «Редактировать».
3. Заполните поля:
   * Дата принятого решения
   * Решение (например, «Удовлетворено», «Отказано», «Частично удовлетворено»)
   * Исполнительный лист (серия/номер в формате XX9999999999 )
   * Дата поступления исполнительного листа
   * Дата окончания исполнительного производства
4. Нажмите «Сохранить».
 
### 6.3. Просмотр списка дел
 
Раздел «Судебные дела» в главном меню показывает все дела по всем должникам. Доступна фильтрация по должнику, датам, решению.
 
 
## 7. Отчёты
 
Доступ: Оператор, Руководитель, Юрист, Администратор.
 
### 7.1. Формирование отчёта
 
1. Перейдите в раздел «Отчёты».
2. Выберите тип отчёта:
   * Статистика переплат на текущую дату
   * Статистика переплат за период
   * Реестр платежей (можно отфильтровать по должнику и периоду)
   * Судебные дела
   * Оборотно-сальдовая ведомость
3. При необходимости укажите период (начало и конец).
4. Нажмите «Сформировать».
5. Отчёт отобразится в рабочей области.
 
### 7.2. Экспорт отчёта
 
1. После формирования отчёта нажмите «Экспорт».
2. Выберите формат: Excel (XLSX), CSV или PDF.
3. Файл автоматически загрузится на ваш компьютер.
 
 
## 8. Администрирование
 
Доступ: только Администратор.
 
Раздел «Администрирование» появляется в главном меню только у пользователя с ролью ROLE_ADMIN.
 
### 8.1. Управление пользователями
 
1. Перейдите в «Администрирование → Пользователи».
2. Вы увидите список пользователей с полями: Логин (доменный), ФИО, Роль, Активен.
3. Добавление пользователя:
   * Нажмите «Добавить пользователя».
   * Укажите доменный логин (например, COMPANY\petrov).
   * Система подтянет ФИО и email из AD.
   * Выберите роль из списка.
   * Установите флаг «Активен».
   * Нажмите «Сохранить».
4. Редактирование: нажмите на строку пользователя → измените роль или статус активности.
5. Блокировка: снимите флаг «Активен» — пользователь не сможет войти.
 
### 8.2. Управление справочниками
 
1. «Администрирование → Справочники».
2. Выберите «Источники переплаты» или «Причины переплаты».
3. Вы можете:
   * Добавить новый элемент (код, название, описание).
   * Редактировать существующий.
   * Деактивировать (мягкое удаление) — элемент не исчезает, но его нельзя выбрать в новых записях.
 
### 8.3. Синхронизация с Active Directory
 
1. «Администрирование → Синхронизация AD».
2. Нажмите «Запустить синхронизацию».
3. Система обновит список пользователей (добавит новых из AD, обновит ФИО и email). Роли существующих пользователей не сбрасываются.
